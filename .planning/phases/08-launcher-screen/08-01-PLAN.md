**Update OnboardingGate in lib/main.dart to add launcher as the first gate:**

The current flow is:
```
onboardingCompleted? → HomeShell : OnboardingFlow
```

Change to a 3-state gate:
```
!launcherCompleted? → LauncherScreen
: !onboardingCompleted? → OnboardingFlow
: → HomeShell
```

**Specific changes to _OnboardingGateState:**

1. Add import at top of file:
   - `import 'screens/launcher_screen.dart';`
   - `import 'services/sample_data_service.dart';` (already has storage_service)

2. Add state variable: `late bool _launcherComplete;`

3. In `initState()`, add: `_launcherComplete = StorageService.instance.launcherCompleted;`

4. Add new handler method:
```dart
void _handleLauncherFinished() {
  setState(() {
    _launcherComplete = true;
    // Re-check onboarding status — demo mode sets it to true via loadDemoData
    _isComplete = StorageService.instance.onboardingCompleted;
  });
}
```

5. Update `build()` method to check launcher first:
```dart
@override
Widget build(BuildContext context) {
  if (!_launcherComplete) {
    return LauncherScreen(onFinished: _handleLauncherFinished);
  }
  if (_isComplete) {
    return const HomeShell(initialIndex: 1);
  }
  return OnboardingFlow(onFinished: _handleOnboardingFinished);
}
```

This ensures:
- Fresh install: launcher → (demo → home) OR (fresh → onboarding → home)
- After demo mode chosen: launcher_completed=true, onboarding_completed=true → straight to HomeShell
- After fresh start chosen: launcher_completed=true, onboarding_completed=false → OnboardingFlow
- Subsequent launches: launcher_completed=true → skips launcher, goes to onboarding gate as before
