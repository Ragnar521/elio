Create `lib/models/weekly_summary.dart` with the WeeklySummary class and manual Hive adapter (typeId: 7).

Follow the same manual adapter pattern used in `lib/models/entry.dart` (no build_runner).

**WeeklySummary fields:**
- `id` (String, UUID) — field index 0
- `weekStart` (DateTime) — field index 1, Monday of the summarized week
- `weekEnd` (DateTime) — field index 2, Sunday end of the summarized week
- `checkInCount` (int) — field index 3, number of entries that week
- `daysWithEntries` (int) — field index 4, unique days with check-ins
- `avgMood` (double) — field index 5, average mood value for the week
- `moodTrend` (String) — field index 6, one of 'up', 'down', 'stable'
- `mostFeltMood` (String) — field index 7, most common mood word
- `bestMoodDay` (String?) — field index 8, e.g. "Thursday" — nullable if no entries
- `bestMoodValue` (double?) — field index 9, the mood value of the best entry
- `bestMoodWord` (String?) — field index 10, mood word of the best entry
- `directionSummaries` (List<Map<String, dynamic>>?) — field index 11, serialized direction data (each map: directionId, title, emoji, weeklyConnections, avgMoodWhenConnected, moodDifference). Nullable for users with no directions.
- `topDirectionId` (String?) — field index 12, direction with highest positive mood impact
- `standoutReflectionAnswers` (List<Map<String, dynamic>>?) — field index 13, serialized reflection highlights (each map: questionText, answer). Max 2 items. Nullable.
- `takeaway` (String) — field index 14, the encouraging message
- `createdAt` (DateTime) — field index 15, when summary was generated
- `viewedAt` (DateTime?) — field index 16, null until user views the full summary

**Computed properties:**
- `bool get hasBeenViewed => viewedAt != null;`
- `String get weekLabel` — returns formatted "Mon D - Mon D" range string (e.g., "Feb 17 - Feb 23")
- `bool get hasDirections => directionSummaries != null && directionSummaries!.isNotEmpty;`
- `bool get hasReflections => standoutReflectionAnswers != null && standoutReflectionAnswers!.isNotEmpty;`

**Why store denormalized data:** Summaries are snapshots. Storing direction titles, emojis, and reflection text directly means the summary remains accurate even if directions are archived or entries edited later. This is a deliberate design choice per RESEARCH.md recommendation.

**Hive adapter notes:**
- Use `reader.readByte()` / `writer.writeByte()` pattern matching Entry adapter
- Handle nullable fields with null checks in read/write
- For `List<Map<String, dynamic>>` fields (11, 13): write as `List<dynamic>` and cast on read. Hive handles Map serialization natively.

**init() method:**

**Core methods to implement:**

1. `Future<bool> hasUnviewedSummary()` — Check if there's a summary for the previous completed week that hasn't been viewed. This is what the Home screen card checks.
   - Calculate previous week's Monday using `_startOfWeek(now).subtract(Duration(days: 7))`
   - Look for existing summary with that weekStart
   - Return true if summary exists and `viewedAt == null`

2. `Future<WeeklySummary?> getOrGenerateCurrentSummary()` — Get the most recent unviewed summary, or generate one for the previous week if none exists.
   - Calculate previous week Monday
   - Check if summary already exists for that week (match by weekStart date)
   - If exists, return it
   - If not, check if there are entries in that week (use StorageService.getEntriesForPeriod)
   - If entries exist, call `_generateSummary(previousWeekStart)` and return result
   - If no entries, return null (no summary for empty weeks — per user decision "always generate summaries regardless of check-in count" BUT this means at least 1 entry needed)

3. `Future<WeeklySummary> _generateSummary(DateTime weekStart)` — Core generation logic:
   - `weekEnd = weekStart.add(Duration(days: 7))`
   - Get entries via `StorageService.instance.getEntriesForPeriod(weekStart, weekEnd)`
   - Get all entries for InsightsService: `StorageService.instance.getAllEntries()`
   - Get insights: `InsightsService.getInsightsForPeriod(now: weekEnd, allEntries: allEntries, period: InsightsPeriod.week, offset: -1, streak: ..., longestStreakAllTime: ...)`
   - Use insights for: avgMood, checkInCount, daysWithEntries, trendUp/trendDown, mostFeltMood
   - Get direction data: loop `DirectionService.instance.getActiveDirections()`, for each get stats and weekly connection count
   - Build directionSummaries list (serialize to Map<String, dynamic>)
   - Find topDirectionId: direction with highest positive moodDifference (>= 0.1 threshold)
   - Get standout reflections via `_selectStandoutReflections(weekEntries)`
   - Generate takeaway via `_generateTakeaway(insights, weekEntries, directionSummaries)`
   - Find best mood entry for bestMoodDay/bestMoodValue/bestMoodWord
   - Create WeeklySummary, save to box, return

4. `Future<List<Map<String, dynamic>>?> _selectStandoutReflections(List<Entry> entries)` — Pick 1-2 standout reflection answers:
   - Collect all reflectionAnswerIds from entries, skip if empty
   - Fetch answers via `ReflectionService.instance.getAnswersByIds(allIds)`
   - Sort by answer length descending (longer = more thoughtful)
   - Pick longest answer as first standout
   - For second: find answer from a different question (by questionText) than the first
   - Return list of maps: `[{questionText: ..., answer: ...}]`, max 2
   - Return null if no reflections exist

5. `String _generateTakeaway(InsightsData insights, List<Entry> entries, List<Map<String, dynamic>>? dirSummaries)` — Generate warm, friend-like takeaway:
   - Build ~12 template patterns based on week data:
     - High mood + streak: "What a week! You showed up {N} days and your mood was on the rise. Keep that momentum going."
     - High mood + reflections: "Your reflections this week show real self-awareness. That's powerful."
     - Improving trend: "Your mood trended upward this week. Something's working — trust it."
     - Stable mood: "Consistency is its own kind of strength. You showed up {N} times this week."
     - Low mood + showed up: "Even on harder weeks, you kept checking in. That takes courage."
     - Low mood + reflections: "Tough weeks are worth reflecting on. You did that — and that matters."
     - Perfect week (7 days): "Seven for seven. You didn't miss a day. That's dedication."
     - One entry only: "One check-in is still a check-in. You showed up, and that counts."
     - Direction engagement: "You connected {N} entries to your directions this week. That's intentional living."
     - Best day reference: "Your best mood was {bestDay} when you felt {bestMood}. What made that day special?"
     - General encouraging: "Another week, another set of data points about you. Keep going."
     - Reflection-heavy: "You reflected on {N} days this week. Self-awareness is a superpower."
   - Selection logic: Match the most specific template first (perfect week > trend-based > general)
   - Adapt tone: if avgMood < 0.35, use gentle/acknowledging templates (no forced positivity). If avgMood >= 0.6, use celebratory templates.
   - Reference specific moments when possible (per user decision: "Reference specific moments from the week")

6. `Future<void> markAsViewed(String summaryId)` — Set viewedAt to DateTime.now(), save to box.

7. `List<WeeklySummary> getAllSummaries()` — Return all summaries sorted by weekStart descending (most recent first). For Insights history browsing.

8. `WeeklySummary? getSummaryForWeek(DateTime weekStart)` — Find by matching weekStart date (compare year/month/day only).

**Helper:**
