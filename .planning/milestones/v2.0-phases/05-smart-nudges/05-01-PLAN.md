**lib/models/nudge.dart** — Create a simple data class (no Hive adapter needed, nudges are transient):

enum NudgeType { dormantDirection, moodPattern, streakCelebration }

class Nudge {
  final String id;

  final String? directionId; // For navigation to DirectionDetailScreen

    required this.type,
    required this.message,

  });

**lib/services/nudge_service.dart** — Singleton service with these methods:

1. **`checkOnAppOpen()`** → `Future<Nudge?>` — Evaluates dormant direction nudges only.
   - Call `DirectionService.instance.getDormantDirections()` to get directions with no connections in 7+ days
   - If any exist and not on cooldown, return nudge for the first one
   - Message format: "It's been a while since you connected with {emoji} {title}. Still on your mind?"
   - actionText: "Reconnect →", directionId: direction.id
   - Cooldown key: `dormant_${direction.id}`

2. **`checkPostCheckIn(int currentStreak)`** → `Future<Nudge?>` — Evaluates streak milestones and mood patterns.
   - **Streak milestones (highest priority, check first):**
     - const milestones = [3, 7, 14, 30, 60, 100]
     - If `milestones.contains(currentStreak)` and not on cooldown:
     - Messages (warm, brief, no exclamation marks):
       - 3: "3 days in a row. You're building something."
       - 7: "A full week. You showed up."
       - 14: "Two weeks of showing up. Consistency matters."
       - 30: "30 days. This is becoming a practice."
       - 60: "60 days of showing up. You're here."
       - 100: "100 days. You've built something real."
     - Cooldown key: `streak_${currentStreak}` (each milestone only once per cooldown)
     - No actionText, no directionId (dismiss only)

   - **Mood patterns (if no streak nudge):**
     - Get entries for last 14 days via `StorageService.instance.getEntriesForPeriod()`
     - Need at least 7 entries to detect patterns
     - Reimplement day-of-week pattern detection (since InsightsService methods are private):
       - Group entries by weekday (DateTime.weekday, 1=Mon to 7=Sun)
       - Calculate average mood per weekday
       - Find overall average across all entries
       - Best day: highest avg, must be >= 15% above overall avg
       - Worst day: lowest avg, must be >= 15% below overall avg
     - For **positive patterns** (best day found), use numbers: "Your mood is {X}% higher on {DayName}s. What makes them work?"
     - For **negative patterns** (worst day found, no best), use gentle language: "{DayName}s seem harder lately. Consider planning something gentle."
     - Cooldown key: `mood_pattern`
     - No actionText, no directionId

3. **`setPendingNudge(Nudge nudge)`** — Store a nudge for Home screen to pick up after returning from check-in flow.
   - Store in instance variable `_pendingNudge`

4. **`consumePendingNudge()`** → `Nudge?` — Returns and clears pending nudge. Called by Home screen on mount/resume.

5. **`dismissNudge(String cooldownKey)`** → `Future<void>` — Store dismissal timestamp in Hive settings box.
   - Key format: `nudge_dismissed_{cooldownKey}`
   - Value: `DateTime.now().millisecondsSinceEpoch` (int, Hive-safe)

6. **`_isOnCooldown(String cooldownKey)`** → `bool` — Check if cooldown period has elapsed.
   - Read from Hive settings box: `nudge_dismissed_{cooldownKey}`
   - Dormant direction cooldown: 7 days
   - Streak milestone cooldown: 30 days (each milestone independent)
   - Mood pattern cooldown: 14 days
   - Determine cooldown days based on key prefix: starts with "dormant_" → 7, starts with "streak_" → 30, else → 14

7. **Helper: `_dayName(int weekday)`** — Convert DateTime.weekday int to name string.

**Concurrency guard:** Add `bool _isChecking = false` flag. Both `checkOnAppOpen` and `checkPostCheckIn` should return null immediately if `_isChecking` is true, and wrap their body in try/finally to reset the flag.

**Settings box access:** Use `Hive.box('settings')` (already opened by StorageService.init()).

**No init() method needed** — NudgeService is stateless except for `_pendingNudge` and reads from already-opened Hive boxes.
