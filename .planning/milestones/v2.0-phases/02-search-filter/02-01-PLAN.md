  List<Entry> filterEntries({

    required EntryFilter filter,
  }) { ... }
}

**Keyword search** (SRCH-01):
- Case-insensitive `String.contains()` matching
- Search across `entry.intention` first
- If no intention match AND entry has `reflectionAnswerIds`, search reflection answer text using `ReflectionService.instance.getAnswersByIds()`
- CRITICAL: Pre-load all reflection answers into a `Map<String, List<ReflectionAnswer>>` BEFORE the filter loop to avoid N+1 lookups. Build map once for all entries that have reflectionAnswerIds:
  ```dart
  final answerCache = <String, List<ReflectionAnswer>>{};
  if (filter.searchQuery != null) {
    for (final entry in entries) {
      if (entry.reflectionAnswerIds != null && entry.reflectionAnswerIds!.isNotEmpty) {
        answerCache[entry.id] = ReflectionService.instance.getAnswersByIds(entry.reflectionAnswerIds!);
      }
    }
  }
  ```

**Mood range filter** (SRCH-02):
- Use `MoodRange.matches(entry.moodValue)` from the model
- Entry matches if ANY selected mood range matches (OR logic)
- Empty set means no filter (show all)

**Date range filter** (SRCH-03):
- Start date is inclusive: `!entry.createdAt.isBefore(dateRange.start)`
- End date is inclusive of the full day: `entry.createdAt.isBefore(dateRange.end.add(Duration(days: 1)))`
- This prevents the off-by-one pitfall from research

**Direction filter** (SRCH-04):
- Use `DirectionService.instance.getConnectedEntries(directionId)` to get connected entry IDs
- NOTE: This is async. Handle by making `filterEntries` async OR by accepting a pre-fetched set of connected entry IDs. Since the research recommends synchronous filtering, accept an optional `Set<String>? connectedEntryIds` parameter that the caller pre-fetches. Add a helper method:
  ```dart
  Future<Set<String>> getConnectedEntryIds(String directionId) async {
    final entries = await DirectionService.instance.getConnectedEntries(directionId);
    return entries.map((e) => e.id).toSet();
  }
  ```
  Then the main `filterEntries` method stays synchronous, using the pre-fetched set.

**Combined filters** (SRCH-05):
- Apply all non-null filters sequentially (AND logic between filter types)
- Each filter narrows the previous result
